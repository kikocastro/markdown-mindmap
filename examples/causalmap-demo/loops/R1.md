label: Firefighting trap
kind: reinforcing
status: active

The core trap: `defects → firefighting →(−) improvement-capacity → test-coverage →(−) defects`.
Firefighting eats the capacity that would raise coverage, so more defects slip. Two `−` edges,
even parity, reinforcing. Run backwards it is virtuous: protect capacity and the same loop
pushes defects down.
